Installation

Please download Gʀᴏᴏᴛ using Pɪᴩ.

pip install groot

You can create an alias to make launching Gʀᴏᴏᴛ easier:

alias groot="python -m groot"

Once the alias is created, you can start Gʀᴏᴏᴛ in Command Line Interactive (CLI) mode:

$   groot

Or in Graphical User Interface (GUI) mode:

$   groot "ui gui"

For other modes, please see the MCᴏᴍᴍᴀɴᴅ documentation.

Tutorial

Introduction

Like Bɪᴏ42, Gʀᴏᴏᴛ uses the MCᴏᴍᴍᴀɴᴅ interaction library. This means it can operate from several different perspectives, including a Command Line Interface (CLI) and Graphical User Interface (GUI). For this tutorial, we'll be using the CLI, because it's much easier to explain. When it comes to your own projects, the GUI has the same commands, but provides a much easier way to visually check everything is going smoothly.

Getting started

Make sure Gʀᴏᴏᴛ is started in CLI mode if it isn't already:

$   groot
    >>> Empty model>

Once in Gʀᴏᴏᴛ, type help for help.

$   help
    ECO help
    INF   help________________________________

          You are in command-line mode.
...

Introduction to the sample data

Gʀᴏᴏᴛ comes with a sample library, let's get started by seeing what's available:

>>> Empty model>file.sample
ECO file.sample
INF seqgen
    sera
    simple
    triptych

The triptych sample contains a gene family which has undergone two recombination events "X" and "Y":

    ALPHA      BETA
      │         │
      └────┬────┘ X
           |
         DELTA         GAMMA
           │             │
           └──────┬──────┘ Y
                  |
               EPSILON

The final gene family, EPSILON, therefore looks something like this:

<--5' (----ALPHA----)(----BETA----)(----GAMMA----) 3'-->

Let's pretend we don't already know this, and use Gʀᴏᴏᴛ to analyse the triptych.

Loading the sample

The sample command can be used to load the sample files automatically, but for sake of this tutorial, we will be loading the data manually.

Unless you can remember where you installed the files to earlier, you can find out where the example is located by executing the following command:

$   sample triptych +view
    ECO sample name=triptych view=True
    INF import_directory "/Users/martinrusilowicz/work/apps/legoalign/legoalign/sampledata/triptych"

That import_directory line tells us where the sample lives. Your path will look different to mine.

You can now load the files into Gʀᴏᴏᴛ:

$   import.blast /blah/blah/blah/triptych/triptych.blast
$   import.fasta /blah/blah/blah/triptych/triptych.fasta

You should notice at this point the prompt changes from Empty model to Unsaved model.

We've not done much so far, but let's save our model with a better name:

$   file.save tri
    ECO file.save file_name=tri
    PRG  │ file_save...
    PRG  │ -Saving file...
    INF Saved model: /Users/martinrusilowicz/.mcommand-data/groot/sessions/tri.groot

We didn't specify a path, or an extension, so you'll notice Gʀᴏᴏᴛ has added them for us.
